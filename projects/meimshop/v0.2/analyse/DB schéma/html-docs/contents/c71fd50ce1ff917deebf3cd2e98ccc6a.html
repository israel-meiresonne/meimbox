Description

Contraintes

Les boxProduct et les basketProduit ont des gestions de stock de tailles différentes:

-BoxProduct: -> Le véritable stock est dans la table BoxProducts-Sizes -> Les stock de la table Product-size indique la répartition des tailles et non la quantité réel cad les tailles dans lesquels le produit peut se décliner

Contraintes

À chaque commande il faut update les tables BoxProducts-Sizes et Products-Sizes

-> pour chaque Boxproduct commandé:
1. réduire le stock à la taille commandé dans BoxProducts-Sizes 2. corriger la répartition de la taille du produit dans la table Products-Sizes

-> pour chaque basket commandé: 1. réduire le stock dans la table Products-Sizes
